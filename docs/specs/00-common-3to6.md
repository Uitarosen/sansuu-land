# さんすうランド v3.0 拡張仕様 — 共通基盤(3〜6年)

現行実装(仕様書 v2.0 = 小1・小2)を前提に、3〜6年生の全単元を追加するための共通仕様。
各学年の詳細は `01-grade3.md`〜`04-grade6.md` を参照。

## 1. 学年構成とワールド

学年が上がるにつれ「かわいい」→「かっこいい・知的」へトーンを移行する。
高学年(11〜12歳)にスイーツ×パステルは幼く映るため、世界観は保ちつつ題材と配色を成長させる。

| 学年 | ワールド | テーマの必然性 | textStyle | 相棒キャラ |
|---|---|---|---|---|
| 3年 | おかし工場 🏭 | 仕分け=わり算、箱詰め=かけ算、検品=はかり | 全ふりがな | ロボットのカカオくん |
| 4年 | スイーツ遊園地 🎡 | 観覧車=角度、コースター=折れ線グラフ、園内マップ=面積 | 難読語のみふりがな | ペンギンのソフトくん |
| 5年 | スイーツ研究所 🔬 | 濃度=割合、計測=平均、水そう=体積、実験=データ | ふりがなオフ標準(設定でオン) | ふくろうのドクターオウル |
| 6年 | おかし宇宙ステーション 🚀 | 燃料=比例、星座=対称、航路=場合の数、卒業=地球帰還 | ふりがなオフ標準 | 宇宙ねこのコメット |

- 配色: 3・4年は現行パステル、5・6年はミント/ネイビー基調の「ラボ/宇宙」スキン(Tailwind テーマ拡張)
- みみちゃんは全学年の案内役として続投。学年相棒はきせかえ・結果画面に登場

## 2. 画面の変更

## 3. エンジン拡張(最優先で実装)

### 3.1 checkAnswer の拡張
- **分数** `'3/4'` 形式。等価分数(6/8)は正解扱い+「もっと かんたんに できるよ → 3/4」バナーで約分を促す
- **複数欄回答**: answer をオブジェクト対応に拡張
  - 商とあまり `{ q: 3, r: 2 }`
  - 帯分数 `{ whole: 1, num: 2, den: 3 }`
- **小数**: Number 比較で `0.30 = 0.3` を許容

### 3.2 Numpad の拡張(mode プロパティで1コンポーネントに統合)
| mode | 内容 | 初出 |
|---|---|---|
| `int` | 現行 | — |
| `decimal` | 小数点キー追加 | 3年 小数 |
| `quotRem` | 商・あまりの2欄(タップで入力先切替) | 3年 あまりのあるわり算 |
| `fraction` | 分子/分母の2欄 | 3年 分数 |
| `mixed` | 整数部+分子/分母の3欄 | 4年 帯分数 |
| `letter` | x キー付き | 6年 文字と式 |

### 3.3 subSkill の拡張
- 九九ヒートマップ方式を横展開: わり算(段別の逆演算)、わり算筆算のステップ別(たてる/かける/ひく/おろす)、%⇔小数変換
- 弱いサブスキルを重点出題する現行ロジックをそのまま利用

## 5. 出題設計の共通原則

1. **誤答は誤概念ベース**(各学年仕様に誤概念カタログを明記)。ランダム誤答は補完用のみ
2. **文章題は2段階**(しき選び wordExpr → 答え wordAnswer)を全学年で継続。高学年は「立式」の比重を上げる
3. **山場単元には操作型を最低1つ**。概念導入直後(difficulty 2)に配置し、手で動かしてから記号操作へ
4. **公式は「体験→発見→公式化」の順**。公式暗記から入るレッスンを作らない
5. **テスト**: 現行 generators.test.js の全単元ループが新学年を自動カバー。学年固有の制約テスト(あまり<わる数、通分の正しさ、誤答が正解と衝突しない等)を追加

## 6. 楽しさ・継続(共通追加)

- 単元マスターで**ワールドが育つ**: 工場の機械/遊園地のアトラクション/実験器具/宇宙船モジュールを獲得し、ホーム画面のビジュアルが変化
- 高学年オプション: **タイムアタック**(計算系単元のみ、自己ベスト比較。他者順位は出さない)
- 6年最終単元クリアで**卒業エンディング**(全学年キャラ登場)
